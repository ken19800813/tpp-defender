## 3. 本階段工作記錄

### v0.2.1 - 規則對話框完全修復

**NameError 根因**：`_rule_editor_dialog()` 中 `update_keywords_state()` 引用不存在的 `TEXT_COLOR` → 整個對話框回覆欄和按鈕都沒建出來。
- **修法**：用 `#e8fdfd` 替代；修復 CTkEntry disabled 狀態 insert bug（必須先 normal → 改 → disable）
- **結果**：優先 checkbox 移到最上面、打勾時自動填字、整個對話框完整出現

### v0.2.2 - 抖內高亮

**新功能**：Super Chat 訊息用鵝黃色底高亮。
- **實作**：獨立 DOM 選擇器 + 新訊號 `"SUPERCHAT"` + 聊天記錄加 `superchat_amount` 欄位

### v0.2.3 - 核心 Bug 衝刺

**Bug 1：Windows 卡死**
- **症狀**：貼 `youtube.com/live/URN_tLI4-As` 時永遠等待聊天室
- **根因**：舊 `extract_video_id()` 只用 `split("v=")` 抓 id，遇到 `/live/` 格式會把整串網址當 id
- **修法**：多格式正規表達式（`watch?v=`、`youtu.be/`、`/live/`、`/shorts/`、`/embed/`）

**Bug 2：雙 @@ mention**
- **症狀**：`@@H_Minnie_米妮`（YouTube handle 本身帶 @）
- **修法**：先檢查 `startswith("@")`

**Bug 3：日誌重複訊息**
- **症狀**：同一則訊息在日誌出現兩次
- **根因**：YouTube DOM 虛擬化，同內容會拿到不同 id
- **修法**：內容層級去重（`(author, content)` key + 6 秒窗口）

### v0.2.4 - UX 微調

- **刪除規則對話框**：加「關閉」按鈕（原本無法取消）
- **後台按鈕寬度**：改 `fill="x" expand=True` 平分寬度（原本固定 150px 擁擠）
- **歷史記錄標題**：截斷到 50 字（原本太長蓋住按鈕）

## 6. 開發提醒

### Tkinter 套路

1. **Pack 順序**：`expand=True` 的元件會吃光空間 → 固定高度的控件用 `side="bottom"` 先卡位
2. **CTkEntry disabled**：必須先 normal 才能修改，改完再 disable
3. **ScrolledText 捲軸**：手動呼叫 `style_scrollbar(widget)` 套用深色主題

### YouTube DOM 監控

- DOM 節點虛擬化 → 同內容可能拿到不同 id → 需要內容級去重
- 同步間隔 0.4 秒適中

### 常見坑點

- Playwright 瀏覽器首次下載 ~200MB（正常）
- Google 登入偵測：只有 CDP 接管真實 Chrome 可靠
- Windows `/live/` URL：確認用最新版本
- 冷卻機制全域（10 秒）
- 優先規則啟用時會完全取代其他規則回覆
